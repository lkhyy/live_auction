# 压测报告

> 脚本位置：[load-tests/README.md](../load-tests/README.md)  
> 目标（项目内约定）：HTTP 出价 P99 &lt; 500ms，错误率 &lt; 5%；WebSocket 约 1500 连接稳定。

## 1. 环境说明

| 项 | 说明 |
|----|------|
| 被测服务 | `npm run dev:api`（NestJS，默认 :3000） |
| 数据 | `npm run storage:reset` 后 LIVE 拍品 `00000000-0000-4000-8000-00000000A01` |
| 工具 | [k6](https://k6.io/)（HTTP）、[Artillery](https://www.artillery.io/) 2.x（WS） |

## 2. HTTP 出价压测（k6）

### 2.1 命令

```bash
# PowerShell
$login = Invoke-RestMethod -Method Post -Uri "http://localhost:3000/auth/login" `
  -ContentType "application/json" `
  -Body '{"email":"buyer@example.com","password":"password123"}'
$env:AUTH_TOKEN = $login.accessToken
$env:AUCTION_ID = "00000000-0000-4000-8000-00000000A01"
$env:API_URL = "http://localhost:3000"
k6 run load-tests/k6-bid-load.js
```

```bash
# bash
export API_URL=http://localhost:3000
export AUCTION_ID=00000000-0000-4000-8000-00000000A01
export AUTH_TOKEN=$(curl -s -X POST http://localhost:3000/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"buyer@example.com","password":"password123"}' | jq -r .accessToken)
k6 run load-tests/k6-bid-load.js
```

### 2.2 场景参数

| 参数 | 值 |
|------|-----|
| 到达率 | 200 次/秒 |
| 持续时间 | 60 秒 |
| 预分配 VU | 300，最大 500 |

## 3. WebSocket 连接压测（Artillery）

### 3.1 命令

```bash
npm install -g artillery@2
export AUCTION_ID=00000000-0000-4000-8000-00000000A01
artillery run load-tests/artillery-ws.yml
```

默认约 25 连接/秒 × 60 秒。逼近 1500 并发需提高 `artillery-ws.yml` 中 `arrivalRate` 并注意本机文件描述符上限。

### 3.2 结果记录

| 指标 | 目标 | 实测 | 是否通过 |
|------|------|------|----------|
| 连接建立 | 稳定无大规模失败 | _待填写_ | ☐ |
| 60s 会话保持 | 无明显断连风暴 | _待填写_ | ☐ |

## 4. 结论与建议

- 压测脚本已纳入仓库，可在答辩前于**与演示相同硬件**上复跑并替换 §2.3、§3.2 表格。  
- 生产环境建议在 API 前增加 Nginx 限流，并水平扩展 API + Redis 主从（当前为课程/demo 单机拓扑）。
